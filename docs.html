1. 시스템 개요

ONDA Selfmarketing이란?

ONDA는 AI 기반 네이버 플레이스 순위 관리 서비스입니다. 매장 사장님들의 네이버 플레이스 검색 순위를 자동으로 관리하고, 트래픽 투입부터 리뷰 관리, 결제, 리텐션까지 전체 고객 라이프사이클을 자동화합니다.

핵심 아키텍처

  • 프론트엔드: Pure HTML/CSS/JS (모바일 퍼스트, 375px 기준)
  • 백엔드: Supabase (PostgreSQL + REST API + RPC)
  • 자동화 엔진: EventHub 기반 이벤트 드리븐 아키텍처 (16+ 이벤트 타입)
  • 최적화 알고리즘: Thompson Sampling (밴딧 알고리즘)으로 매체 선택 최적화
  • DB 규모: 52개 테이블, 36개 페이지

EventHub 드리븐 자동화

모든 주요 비즈니스 액션은 이벤트로 기록되고, 연쇄 액션을 자동 트리거합니다. 예를 들어 kakao_friend_verified 이벤트가 발생하면 → 고객 status가 trial로 변경되고 → 체험 시퀀스가 자동 생성되고 → 카카오 시퀀스가 시작됩니다. 총 16개 이상의 이벤트 타입이 등록되어 있습니다.

Thompson Sampling 매체 최적화

트래픽 투입에 사용하는 외부 매체를 선택할 때 Thompson Sampling(밴딧 알고리즘)을 사용합니다. 각 매체의 성공(alpha)/실패(beta) 파라미터를 베타 분포로 모델링하여, 탐색(exploration)과 활용(exploitation)을 자동으로 균형있게 수행합니다.

4. EventHub 이벤트 (16+)

EventHub는 이벤트 기반 자동화 엔진입니다. 이벤트가 발생하면 등록된 연쇄 액션이 자동으로 트리거됩니다. 모든 이벤트는 sm_events 테이블에 기록됩니다.

1. kakao_friend_verified
트리거: 카카오 채널 친구 인증 완료 시
→ 고객 status를 trial로 변경 → 체험 시퀀스(D+1~D+30) 생성 → 카카오 자동 시퀀스 생성
2. trial_applied
트리거: 체험 신청 완료 시
→ KPI 카운터 증가 (sm_kpi_daily trial_count +1)
3. trial_converted
트리거: 체험 → 유료 전환 시
→ 외주 주문 자동 생성 (sm_vendor_orders) → 결제 기록 생성 (sm_payments) → KPI 업데이트 (conversion_rate)
4. onboarding_completed
트리거: 온보딩 완료 시
→ 고객 status를 active로 변경 → 온보딩 부스트 종료
5. churn_risk_detected
트리거: 이탈 위험 점수 60점 이상 감지 시
→ 리마케팅 큐 추가 (sm_remarketing_queue stage 1) → 위험 알림 생성 → 고객 status를 at_risk로 변경
6. payment_failed
트리거: 결제 실패 시
→ 결제 실패 핸들러 RPC 호출 → sm_payment_failures 기록 → 실패 카운트에 따른 자동 대응 (즉시/24h/48h/7일)
7. customer_churned
트리거: 고객 해지(churned) 시
→ 환불금 계산 (잔여 기간 비례) → 외주 주문 중단 → 30일 후 윈백 시퀀스 스케줄
8. message_sent
트리거: 메시지 발송 시
→ 발송 스케줄 업데이트 → 타겟풀 상태 추적 → KPI 발송 카운터 증가
9. referral_converted
트리거: 추천 링크를 통한 가입 전환 시
→ 추천인 상태 업데이트 (sm_referrals status → converted) → 추천인 보상 지급
10. blog_filtered
트리거: 블로그가 네이버에 의해 필터링(비노출) 시
→ 블로그 스냅샷 업데이트 (is_filtered = true) → 품질 저하 알림 → 해당 블로그 투입 중단
11. logic_change_detected
트리거: 네이버 로직 변경 감지 시 (3.5일 주기)
→ 크리티컬 알림 발송 → 48시간 신규 주문 중단 → 기존 주문 효과 재평가
12. source_health_alert
트리거: 특정 매체의 효과 이상 감지 시
→ 리더보드 베타 파라미터 조정 (beta +1) → 해당 매체 사용 비중 자동 감소 → 대체 매체 탐색
13. season_event_started
트리거: 시즌 이벤트(명절, 방학 등) 시작 시
→ 탐지 임계값 조정 (시즌 변동 허용) → 시즌 보상 계수 적용 → N2 평가 기준 보정
14. rollout_phase_advanced
트리거: 롤아웃 단계 진행 시 (phase N → N+1)
→ A/B 테스트 단계 관리 → 대상 고객 비율 확대 (target_pct 증가) → 이전 단계 결과 기록
15. payment_overdue
트리거: 결제일 초과 (연체) 시
→ 대기 중 외주 주문 취소 → 24시간 간격 리마인더 발송 → 7일 후 자동 해지 스케줄
16. vendor_order_completed
트리거: 외주 주문 완료(효과 확인) 시
→ 리더보드 alpha/beta 업데이트 (성공 시 alpha+1, 실패 시 beta+1) → Thompson Sampling 분포 갱신 → 매체 순위 재계산

8. 용어 사전

N2
네이버 플레이스 순위 지표. 검색 키워드별 매장 노출 순위를 수치화한 것. 숫자가 낮을수록 상위 노출.
안티패턴
투입 효과를 저하시키는 비권장 패턴. 예: 주말 집중 투입, 매체 편중(동일 매체 연속 사용), 급격한 볼륨 변화.
Thompson Sampling
베이지안 밴딧 알고리즘. 각 매체의 alpha(성공 횟수)/beta(실패 횟수) 파라미터로 베타 분포를 구성하고, 샘플링을 통해 최적 매체를 선택. 탐색과 활용의 자동 균형.
디케이 프로파일
투입 효과의 시간에 따른 감쇠 모델. 반감기(half-life)로 측정. 예: 반감기 3일 = 투입 후 3일 지나면 효과가 절반으로 감소.
카니발리제이션
동일 업종 내 키워드 간 검색 트래픽 잠식 현상. 예: "강남 맛집"과 "강남역 맛집"이 서로의 트래픽을 빼앗는 것.
교차 효과
한 키워드에 대한 투입이 다른 키워드의 순위에 미치는 영향. 양의 효과(시너지) 또는 음의 효과(간섭)가 있을 수 있음.
밴딧
Multi-armed bandit 문제. 여러 선택지(매체) 중 최적의 것을 찾기 위해 탐색(exploration, 새로운 매체 시도)과 활용(exploitation, 알려진 좋은 매체 사용)을 균형있게 수행하는 문제.
EventHub
이벤트 기반 자동화 엔진. 비즈니스 이벤트 발생 시 등록된 연쇄 액션을 자동 트리거. 16개 이상의 이벤트 타입 지원.
핫리드
전환 가능성이 높은 잠재 고객. 핫리드 점수를 기반으로 우선순위가 매겨지며, 높은 점수의 리드부터 접근.
리마케팅
이탈하거나 미전환된 고객에 대한 재접근 캠페인. 4단계(stage 1→2→3→4)로 점진적으로 접근 강도를 높임.
온보딩 부스트
신규 가입 첫 7일간 강화 투입 프로그램. 일일 180-200 타수 + 리뷰 3건으로 빠른 순위 상승을 유도하여 조기 성공 경험 제공.
이탈 히스테리시스
이탈 판정의 안정성을 위한 메커니즘. 이탈 진입 임계값(60점)과 복구 임계값(40점)을 다르게 설정하여, 경계선 고객의 상태가 빈번히 바뀌는 것(chattering)을 방지.
시즌 보상
시즌 변수(명절, 방학, 연휴 등)로 인한 N2 변동을 보정하는 계수(impact_factor). 시즌에는 검색량이 변하므로 평상시 기준으로 평가하면 왜곡이 발생.
복합 효과
여러 투입(매체, 리뷰, 키워드)의 시너지 또는 간섭 효과. 단순 합산이 아닌 비선형적 상호작용.
롤아웃
신규 기능이나 정책을 전체 고객에 한번에 적용하지 않고 단계적(5%→20%→50%→100%)으로 적용하는 과정. 위험 관리 목적.
파이프라인
고객 상태 흐름 경로. lead → trial → onboarding → active → at_risk → churned → winback. 각 단계 전환 시 EventHub 이벤트 발생.
그레이스 피리어드
결제 실패 후에도 서비스를 유지하는 유예 기간. 이 기간 동안 고객에게 결제 재시도를 요청하며, 기간 만료 시 자동 해지.
매체
트래픽 투입에 사용하는 외부 서비스 소스. 각 매체는 Thompson Sampling으로 효과가 평가되며, 리더보드에서 순위가 매겨짐.
타수
트래픽 투입 단위 (건/일). 예: "3,000타" = 하루 3,000건의 트래픽 투입. 패키지별로 제공되는 타수가 다름.
