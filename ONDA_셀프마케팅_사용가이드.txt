2. 고객 유입 흐름
========================================

고객이 체험 신청하면 자동으로 이렇게 됩니다:

[고객] trial.html 접속
  → STEP 1: 매장 정보 입력
  → STEP 2: 카카오 채널 친구 추가 + 인증코드 전송
  → [관리자] 고객관리에서 💛 버튼 클릭 (인증 처리)
  → 자동: 체험 시퀀스 7개 생성 (D+1,3,5,7,10,14,30)
  → 자동: 카카오 시퀀스 4개 생성 (즉시,24h,72h,7d)
  → 상태: lead → trial

※ 카카오 인증 방법:
  - 고객이 카카오 채널에 인증코드(ONDA-XXXXXX)를 보냄
  - 관리자가 카카오 채널 채팅에서 코드 확인
  - 고객관리 페이지에서 해당 고객 옆 💛 버튼 클릭
  - 체험이 자동으로 시작됨

3. 상태 전환 (자동 연쇄 반응)
========================================

상태를 바꾸면 자동으로 연쇄 반응이 일어납니다:

[lead → trial]
  → 체험 시퀀스 + 카카오 시퀀스 자동 생성

[trial → onboarding] (유료 전환)
  → 결제 기록 자동 생성
  → 외주 주문서 자동 생성 (패키지별 타수)
  → KPI 업데이트

[onboarding → active]
  → KPI 활성 고객 +1

[active → at_risk] (이탈 위험)
  → 리마케팅 큐에 자동 추가
  → 위험 알림 자동 생성
  → KPI 업데이트

[any → churned] (해지)
  → 환불 자동 산출
  → 30일 후 윈백 시퀀스 자동 생성
  → KPI 업데이트

※ admin → "이탈위험 스캔" 버튼으로 전체 고객 일괄 스캔 가능

4. 각 페이지 설명
========================================

【영업】
• 핫리드 (hotleads.html)
  - 87,000개 매장 DB에서 순위 하락 매장 자동 발굴
  - 우선순위 점수로 정렬
  - 클릭 한 번으로 개인화 메시지 생성+복사

• 메시지 템플릿 (messages.html)
  - A: 직접적 제안 / B: 공감형 / C: 시즌 훅
  - 체험 링크 자동 삽입
  - 복사 시 A/B 테스트 자동 기록

• 리마케팅 (remarketing.html)
  - 4단계 퍼널: 알림 → 리마인드 → 특별제안 → 최종
  - 이탈 위험 고객 자동 큐 등록

• 카카오 자동응답 (kakao.html)
  - 즉시 → 24h → 72h → 7d 시퀀스
  - "발송" 버튼으로 처리

• 레퍼럴 (referral.html)
  - 추천인 30% 할인 / 피추천인 50% 할인
  - 코드 생성 → 공유 → 전환 추적
  - 전환 퍼널 시각화

• 커뮤니티 시딩 (community.html)
  - 아이보스/셀클럽/당근/네이버카페 주1회 콘텐츠
  - 4주 로테이션 (로직분석/리워드효율/역분석/안티패턴)

• 타겟 DB (targets.html)
  - 타겟 풀 잔여량 + 고갈 예측
  - M1~M8 발송 볼륨 스케줄 vs 실적

• 대행사 (agency.html)
  - 볼륨 할인: 10개 미만 30% / 10-30개 35% / 30개+ 40%
  - 수익 시뮬레이션
  - 대행사 CRUD

【운영】
• 고객관리 (customers.html)
  - 7단계: 리드 → 체험 → 온보딩 → 활성 → 위험 → 해지 → 복귀
  - 상태별 필터 + 이탈위험 점수 표시
  - 💛 카카오 인증 버튼
  - 📋 이벤트 로그 조회

• 체험 시퀀스 (trial-sequence.html)
  - D+1 순위확인 → D+3 변동알림 → D+5 만료예고 → D+7 전환유도
  - D+10 감쇠경고 → D+14 경쟁리포트 → D+30 로직변경
  - 온보딩 부스트 모드: 180타→150타→AI 최적화

• 외주 포털 (vendor.html)
  - 주문서 자동생성 (전체 활성 고객 대상)
  - 매체/키워드/투입량/미션믹스 표시
  - 대기 → 진행중 → 완료 워크플로우

• 결제 관리 (payments.html)
  - 결제 추가/실패 처리/환불 계산
  - 실패 자동 대응: 즉시 중단 → 24h 알림 → 48h 중단 → 7d 해지
  - 환불 자동 산출: 총결제 - 사용일 비례 - 집행원가

• 대시보드 (dashboard.html)
  - KPI 한눈에: 활성/체험/핫리드/매출/전환율/위험
  - 파이프라인 시각화
  - 최근 활동 로그
  - 시스템 상태 (DB 연결/핫리드/이벤트)

• KPI (kpi.html)
  - 일/주/월별 목표 vs 실적 (M1→M12)
